# 02. ゲームルール

## 1. 基本ルール
- **目的**: スタート地点からゴール地点まで最速で到達する。
- **ゲーム形式**: タイムアタック

## 2. 競争要素
- **CPUとの競争（リアルタイム並走）**: 
  - ゲーム中、プレイヤーと同じコースをCPUキャラクターがリアルタイムに一緒に走る。
  - **当たり判定なし**: プレイヤーとCPUが衝突（クラッシュ）して進行の妨げになること（当たり判定）はない。
  - **強さ（難易度調整）**: CPUの走る速度や強さは、プレイヤーのレベル（過去の実力や進行ステージ等）に合わせて自動で調整される。これにより、常にデッドヒートの緊張感を味わえる。
- **他プレイヤーとの競争（非同期マルチプレイ）**:
  - 他のプレイヤーが記録したタイムと競う形式（※ゴースト表示機能はなし）。
  - 各ステージ終了後に算出される、ネットワーク上のリーダーボード（ランキング）機能を利用して競争する。

## 3. サーバー連携とデータ同期
- **ゲスト登録システム**:
  - 面倒なID、パスワード、メールアドレスの登録は不要。「プレイヤー名（名前）」を入力・登録するだけで即座にプレイ可能なゲスト登録方式。
- **サーバーの役割**:
  - **タイムデータの保存**: プレイヤーがステージをクリアした際のクリアタイムを、登録した「プレイヤー名」と紐付けてサーバーに送信・記録する。
  - **ランキング同期**: サーバーに保存された全プレイヤーのベストタイムデータを取得し、ゲーム内の「タイム記録」「タイム、ランキング」画面で最新ランキングを同期・反映する。
  - **ランキング表示名**: ランキング画面には、各プレイヤーが登録した「プレイヤー名」がタイムと共に表示される。
  - **データの永続化**: プレイヤーの登録名、個人ベスト記録、ステージのクリア状況（アンロック等）を端末に紐付けてサーバー側に保存・同期する。
